Generated by running the genuine R `admixtools` package on the real AADR data — no

## Internal pipeline that produces X / Q (matches the steppe design)

```
read_f2(f2_dir)
  -> admixtools:::f2blocks_to_f4blocks(f2_blocks, left, right)   # -> X blocks
  -> admixtools:::f4blocks_to_f4stats(., boot=FALSE)$est  ($X)   # f4 estimate vector
                                                  $var  ($Q)     # jackknife covariance
qpadm then fudges: diag(Q) += fudge * tr(Q)  before inverting.
```

### Canonical-invocation caveat (important for M(fit-1))
The values are bit-exact **only** through `qpadm(f2_dir, ...)` (qpadm reads the f2 blocks
itself, with its own internal read-arg set). Calling `read_f2(dir)` first and then
`qpadm(f2_blocks, ...)` with non-default read args yields a ~1e-5 difference. When the
steppe M(fit-1) parity test regenerates from f2 blocks, mirror the directory-path form
(or the exact `read_f2` args qpadm uses internally) to hit bit parity.
